Rationale

Discord's streaming tools aren't adequate for certain circumstances which prevents it from being as idiot-proof as it might like. While convenient and low-latency, its no-config nature can lead to headaches and the video quality can be sub-par, even with a Discord Nitro subscription. Luckily, we have a solution in the form of a custom fork of OBS which includes a WebRTC plugin and Broadcast Box, the backend to support this.
Something's in OBS aren't perfect either, so I've included some plugins to make streaming a bit easier, like win-capture-audio to capture application's audio directly, bypassing the need for VoiceMeter Banana or any other external tools to route audio around voice chat or other situations. Go through the list below and see all the added functionality.

i) Install

To install rOBS, simply unzip rOBS.7z to anywhere! (For NDI support, run the included NDI 5.5.3 Runtime.exe inside the top directory.)
To run rOBS, open bin\64bit\obs64.exe Create a shortcut to this destination if you'd like.

ii) Configure

You can use rOBS like your normal OBS install, it might even port over your existing scenes from another install of OBS. For our uses however, we will be taking advantage of someone's Broadcast Box instance. I'll be spinning up my own soon™.
Use these settings:

    File > Settings > Stream
    Service to WHIP
    Server to https://b.siobud.com/api/whip
    Bearer Token to anything you want. Something long so others are unlikely to spy.
We'll leave the quality settings alone until we can properly stress test the servers.

iii) Viewing

This is the easy part! Go to b.siobud.com, and enter in the streamkey. Finito.
